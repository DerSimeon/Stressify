# Contributing to Stressify

Thank you for contributing.

Stressify is a developer tool intended for plugin testing and controlled load testing.
Contributions that enable malicious use will not be accepted.

## Guidelines

- Keep changes focused and minimal
- Prefer clarity over cleverness
- Avoid unnecessary abstractions

## Code style

- Idiomatic Kotlin
- Explicit concurrency
- Predictable lifecycle handling

## Pull requests

- Use descriptive commit messages
- One logical change per PR
- Open an issue first for large changes

All contributions are released under the MIT License.
